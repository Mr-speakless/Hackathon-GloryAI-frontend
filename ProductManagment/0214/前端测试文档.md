# GloryAI 前端测试文档（简版）

## 1. 测试目标
- 验证当前原型主流程可用：`upload -> analyzing -> report -> 重新检测`
- 验证上传校验、Mock 数据展示、页面跳转是否正常

## 2. 测试环境
- 项目路径：`/mnt/e/设计项目/GloryAI/GloryAI_frontend`
- 启动命令：
```bash
cd /mnt/e/设计项目/GloryAI/GloryAI_frontend
npm run dev -- --host
```
- 访问地址：`http://localhost:5173`

## 3. 前置条件
- Node / npm 已安装
- 依赖已安装（`npm install` 已执行）
- `src/services/api.js` 中 `USE_MOCK = true`

## 4. 测试用例（手动）

### TC-01 页面首次加载
- 步骤：打开首页
- 预期：
  - 显示上传页
  - 左侧文案、右侧上传区域可见

### TC-02 上传 JPG/PNG 成功
- 步骤：上传一张 JPG 或 PNG 图片
- 预期：
  - 显示预览图
  - 无报错文案

### TC-03 上传非法格式失败
- 步骤：上传非 JPG/PNG 文件（如 `.txt`）
- 预期：
  - 显示“仅支持 JPG / PNG”
  - 不进入分析流程

### TC-04 上传超大图片失败
- 步骤：上传大于 10MB 的图片
- 预期：
  - 显示“图片不能超过 10MB”
  - 不进入分析流程

### TC-05 分析流程
- 步骤：上传合法图片后点击“开始分析”
- 预期：
  - 进入 loading 页面
  - 约 2 秒后自动进入报告页

### TC-06 报告页内容
- 步骤：观察报告页
- 预期：
  - 左侧显示上传图片
  - 右侧显示 Report 区块
  - 显示皮肤类型、Top 问题、推荐产品卡片

### TC-07 重新检测
- 步骤：点击“重新检测”
- 预期：
  - 回到上传页
  - 可再次上传并分析

## 5. 回归重点（每次改动后至少跑一次）
- TC-02（合法上传）
- TC-05（状态切换）
- TC-07（重置流程）
